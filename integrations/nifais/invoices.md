The first screen presented to the user provides  different  ways  to  search
for a purchaser to produce an invoice. It also provides menu options at  the
top left of the screen that enable  the  user  to  configure  the  way  they
produce invoices.

The user may search for the computer number to bring up the invoice.
[pic]

The user may leave the purchaser name or number blank to bring up  everybody
who has bought.
[pic]

The user may enter just the first letter of the purchaser surname  to  bring
back all purchaser whose surname begins with that letter.
[pic]

The user may enter the surname
[pic]

The user enters search criteria in any of the above  ways  and  presses  the
enter key. This will provide a list of matches if there are  move  than  one
possible choice, or if there is only one  choice  take  the  user  into  the
invoice screen.
[pic]

The user can select from the list either by double  clicking  the  row  they
want to deal with or entering the Reference Number shown on  the  left  hand
side of the row in the text box at the bottom of the screen,
[pic]

The main invoice page will be displayed.
[pic]

The user will see the purchasers details at the top left of the  screen  and
the lot details beneath, with the sections traded in  shown  top  right  and
the totals just below that.

The bottom of the screen shows any average weights and prices  and  provides
all options available to listing, invoicing, transferring, etc.
[pic]

Transferring Lots.
To transfer a lot the user must either double click the  row  they  want  to
transfer and an X marker will be placed in the T column to show  it  is  due
to be transferred, or move the arrow keys and press the space bar button  to
mark with an x.
[pic]

The user must then enter T to transfer in the Action textbox.  If  all  lots
where to be transferred it is not necessary to mark them  with  an  x,  just
use the T to transfer option.
[pic]

The user must select who the lots are to be transferred to,  by  entering  a
customer number or name. The user is asked to confirm the new purchaser.

Invoicing
To produce an invoice enter I in the action  text  box.  The  user  is  then
prompted
[pic]

If there are deductions or alterations to be made to  the  invoice  enter  D
and press enter.
[pic]

Select the ref number of the charge you intend  to  make  and  press  enter.
Enter the amount of the charge and ensure you prefix it with either a  +  or
– to determine whether it is a charge or a discount. Press F5  function  key
to save or click the F5 – Save button.

If you leave the invoice at this point having entered a charge you  will  be
asked if you wish to keep these alterations or discard them.
[pic]

If you proceed with the invoice and the purchaser is making  a  payment  you
will see this form.
[pic]

The user will see what the purchaser owes and has a number of  choices  that
can be made depending on the options set up in the  options  menu.  If  they
want to pay off todays debts the user can enter  a  T  or  in  this  example
could enter reference number 1. They are then asked if they want to pay  all
of it. Pressing enter will pay all or it.  Entering  a  part  payment  value
will part pay it.

They are then asked what method of payment is being used. This again may  be
set up in different ways depending upon the payment  options  the  user  has
already setup in the Options Menu.
[pic]

The user selects 1 or 2 and then either presses the  enter  key  to  confirm
the full amount is to be paid by that method or  enters  the  value  of  the
payment by that method and then selects another method and enters the  value
of that method.

Movement Details.
For some livestock the user will be asked to enter movement details  and  be
able to enter haulier details also. The date may be amended if required  and
the user is shown whether a sheep licence will be printed or not.
[pic]

The holding details will be filled  in  if  they  are  entered  against  the
purchasers account. If they are not the user must enter them manually.  They
may not be left empty.

[pic]

When an invoice has been run and a search is done the status  will  show  as
invoiced.
[pic]

Cancel An Invoice
To cancel an invoice the user brings up the invoice they wish to cancel  and
enters C into the action text  box.  They  are  then  presented  the  cancel
screen
[pic]

In the example there is only one invoice but there can be  more.  Enter  the
reference number to cancel and press the  enter  key  or  click  the  cancel
invoice button.
[pic]

Confirm the invoice to be cancelled.
[pic]

The invoice is cancelled and the action is confirmed.
[pic]

When the user is in the main invoice screen invoiced lots are shown by an  I
in the invoice column (I?).

Options Menu

The options menu sets options per computer. Every computer  or  laptop  that
runs invoices must be set up to run the options you require.  It  is  up  to
the user to check these and set these up.

Sort Lots By
This option does what it says on the tin so to speak. Tick  the  option  you
require to be the default. If you want the lots  to  be  displayed  as  they
have been entered into the system select do not sort lots. If you want  them
sorted in lot order select by lot number. If you produce  one  invoice  with
different sections and want to sort by section and then  lot  number  select
this option. You may either have a screen pop up  and  remind  you  to  sort
lots and show you these options or if you  do  not  want  this  then  untick
display sort option.

Printers
You must set up which  printers  you  will  be  printing  various  sorts  of
invoice paper work out on. You can print to  any  printer  on  the  network.
This option enables  you  to  select  a  printer  with  pre-printed  invoice
stationary in it to print invoices  and  plain  paper  to  print  the  sheep
licence or list, or you may deceide to print the invoice and licence on  the
same staiotnary. The choice is yours.

[pic]

Check Every
The user can set up how often they are reminded to check they  have  entered
the correct invoice number to correspond to the invoice number on  the  pre-
printed stationary.
[pic]

Display Only Uninvoiced Purchasers
This option may be used at any time  to  see  which  buyers  have  not  been
invoiced. Select this option and leave the search criteria empty  and  press
enter to see a list of everybody who  had  bought  but  not  been  invoiced.
Remember to take this option off again to see everyone  who  has  bought  on
the day.

Email BCMS Purchaser Movement.
This is not the on/off market movement. This is only needed if  you  act  as
an agent for the purchaser and record the on to their holding movement.

Print Sheep Licence By Default
This option will set the flag as Y for yes to print a  sheep  licence  after
the invoice is produced.

Show Full Descriptions
Some sales such as pedigree, farm, dairy, furniture or fineart  and  antique
sales reuire longer descriptions than usual livestock sales. For sales  like
these it may be required that the entire long description  is  printed  upon
the invoice. If this is so mark this option with a tick.

Display Vendor Name
To show the vendor name on the invoice tick this option.

Payment Options

Display Old Debt Details
This option  if  ticked  will  show  any  debts  still  outstanding  on  the
purchaser account form previous sales.

Print Paid Detail
This option will show the payment method that was used to pay the invoice.

Simple Payments
Not all customers want to record the method by which the debt was  paid  but
simply want to record that it was paid. These  customers  should  tick  this
option.

Don’t Pay Old Debts
This option ensure that old debts are not shown.

Will do as it imples.

Print Date of Birth
This option is clearly for livestock and if a DOB  is  entered  against  the

and the sub accounts are cross referenced to it so that all  debts  for  the
purchaser are shown on the same account number on the debtors  list.  Select
the main account number option. Enter the sub account and then put the  main
account number in against it.
