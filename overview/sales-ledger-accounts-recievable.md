After Sales

Sales Ledger

The sales ledger is where the debtor records aka accounts recievables are stored.

Transaction Input			-	Manual input of credit or debit into the ledger

Transaction Edit			-	Edit a credit or debit in the ledger

Transaction Payment			-	Pay off a credit or debit in the ledger

Buyers Statement & Interest		-	Add interest and produce statements

See Buyers Record			-	Lookup a customers account

Print Aged List of Debtors		-	Report of Debtors by period

Print Simple List of Debtors		-	Report of all debtors

Debtor Totals by Office			-	Report of debts by Office/Sale/Dept

Analyse Purchase Transactions		-	Report of all credits & debits in a time

Update Sales Ledger from Sage		-	Import ledger info from Sage Accounting

Print Copy Payment Daybooks		-	Reproduce list of payments made in the ledger

Archive Sales Ledger Transactions	-	Archive old debts

Maximum Balance Report		-	Report of trading history & highest debt level

Synchronise Customer Index		-	Utility to check

Transaction Input

Description: Is designed to allow users to load tranasctions against customer ledger account.

Enter Customer name or number and press enter

Move up and down the grid with the arrow keys, page up and page down, click on a customer with your mouse or enter the account number in the Acc No text box.

Enter Date (the system will default to todays date), Office number from list in the middle of the screen,

Section number from list in the middle of the screen, value, comments and press enter to save or click on the save button

The system will go back to allow the user to enter the next customer, repeat the same procedure

As more inputs are loaded a batch detail form will populate with  information.

When finished Esc/F8 keys will return the user to exportable/printable report with all the payments on. It also creates a CSV file that can be exported into Sage Accounting.

Transaction Edit

This option is now in ‘See Buyers Record’ under the options menu.

Transaction Payment

Is designed to allow users to pay off transactions against a customer. Example; PC Jones owes £1000 for a sale last month and has sent in payment of £1000 to pay off the debt.

Enter the name of the customer to pay off a transaction. By defualt the system will show ‘Outstanding Debtors Only’.

Enter the payment date (by default the system will have today’s date),  select the payment type from the drop down list. Enter the payment amount, add a comment if required

Enter the record number to pay off or enter ‘G’ for General to reuce the overall amount of debt. Click on save or press F5 key.

The debt will reduce to zero and be ticked off.

When the user has finished making transaction payments a report is generated that is printable/exportable showing all payments made.

Options

The user can setup a receipt to be produced when a transaction is paid off

Buyers Statement & Interest

Calculate Interest

Enter the vale of interest that is to be applied for a given period

View & Post Interest Calculated

Displays the interest that has been accrued by customers, if no interest has been accrued the following form appears

The option will allow the user to view the interst and post it to the ledger.

Print Buyer Statements

Enter the relevant dates for the period the statements cover. Either type in the required dates or click on the icon of the calandar

Double click on the required dates, by default today will be selected.

Buyers statements can be further filtered by office and mailing groups (see contact managament for full use of mailing groups).

Select All accounts for a statement fun or individual to select a single customer to produce a statement for. If ‘All Accounts’ is selected the user can filter the printout by customer surname, e.g. A-Z

If the user selects ‘Individual Account’ the user is prompted to search for the customer by name or number.

The statement is displayed and can be printed or exported

See Buyers Record

Is designed to allow users to view outtsanding balances due and all debt history

Select the customer to view their record in the sales ledger

At a glance; the customers name, address & phone number are displayed. The customers accountnumber, account balance and credit limit are also displayed. If any notes exist against the customer these can also be shown or hidden by F11 button. Double clicking on the name will load the customer view/amend screen.

The grid will display all outstanding transactions, including reference number, date, office, comments, purchase amount, paid amount, balance (overall balance for the customers account) & transaction balance (balance outstanding on the transaction in question.

The user can filter the view of the buyers record by selecting ‘All Debt History’, selecting a record number to display from or displaying archived records.

Options

The user can print or export the buyers record and can choose from an internal office report or an external statement print that can be given to the customer.

The user can edit a transaction by either clicking on the options menu and selecting ‘Edit Highlighted Trasaction’ or by right clciking on the transaction in the grid.

The user can edit the following; date, office, section & comments.

The user can select printers for; Payment Receipt, Daybook, Cheque & Report.

Statements can be configured in the following way

The user can setup; a header and footer gap for pre-printed stationary, age debt weekly, include the office on the statement, create an overdue message and a trigger for the number of days before it becomes active.

The user can also add and view comments made against the customer in the ‘See Buyers Record’ screen. See Contact Management for details on the Contacts suite.

Aged List of Debtors

Description: Is designed to allow users to create filtered reports of debtors by date.

The user can enter a date to age the debts from , by default today will be selected. The report can be further filtered by; ‘Balances From’, mailing groups and offices. The user can select the customers A-Z to filter the report. The report can be configured as; weekly, monthly or quarterly & can be ordered by customers initials or address.

By double clicking on a customers name it will drill down to their buyers statement, from there each transaction can be double clicked on and will load a copy invoice if appropriate.

The user can print or export the report by clicking on the file menu;

The printouts are as follows; summary, detailed or over credit limit.

Simple List of Debtors

Description: is designed to allow users to prodice a report of their debt list

The report can be printed or exported.

Double clicking on an account will drill down to the buyers record.

The ‘Age By’ menu allows the user to age the debt of the report by week, month, quarter.

Debtor Totals By Office

Description: Is designed to filter debtors by office

The report can be printed or exported. Double click on the office to see a detailed breakdown of the debtors of that office.

Options:

The user can select options and ‘View Debts By Date’

Double click on the a date and it will list all debts from that date.

The report can be printed or exported.

Update Sales Ledger from Sage

Description – Is designed to allow the import of a file from Sage into the Newline system

Click on the ‘Find Sage File’ button and browse to the relevant file.

Print Copy Payment Daybooks

Description – Is designed to allow the re-printing of daybooks that have been generated when paying old debts.

Enter the year, batch number or date of transaction payment.

Double click on the batch number required and click on the ‘View’ button.

The report will list the customer, the value paid, types of payment and the customer balance. The report can be printed by clicking on the ‘Print’ button.

Archive Sales Ledger Transactions

Description – routine to archive paid transactions to improve search performance of the ledger

Maximum Balance Report

Description – Generates a report of outstanding debts and a maximum value outstanding during a given period

Enter a ‘From’ date & ‘To’ date click on Continue to generate the report.

The report can be printed or exported.
