History Analysis

End of Sale Reports

Description – A series of reports generated when closing a sale that can be viewed or reprinted at any time

Click on the required year and select the relevant sale from the list. A series of reports are available to view and print. Either select the reports required or by default select all reports and click the ‘Next’ buton.

Reports can be printed.

Turnover Analysis

View Sale Analysis

Description – Is designed to generate a turnover report comparing a user defined period to the year to date

Enter a ‘Start’ and ‘End’ date, click on the ‘Continue’ button. Filter the report by selecting ‘Vew by Sale or Office’.

Select the required Office, year and Sale and click the ‘Continue’ button’. The report can be printed.

View Commission Allowance

Description – Is designed to allow users to group offices into heading that can compare commission against the same periodfrom last year.

Setup

Select the ‘Settings’ menu and click on ‘Office Groupings’.

Click on ‘Add New Header’ and type in a description for the new header. Then click on the header in the ‘Groupings Headers’ grid. Select the offices one by one for that header and click on ‘Add to Selected Group’. Click on ‘Save’ when finished.

Next; click on Settings  and ‘Financial Year Start Dates’ and select the required date.

Click on Settings and ‘Commission Charges’. Select the charges and deductions that make up commission and click ‘Save’ when finished.

Enter a ‘From’ date and ‘To’ date.

Select the turnover sections that are required in the report & click ‘Continue’.

The report is printable and exportable.

Seller / Buyer Trading Analysis

Description – Is designed to allow users to analyse data & trading trends through user definable date ranges.

Select analysis by Seller, Buyer or Customer (customer will only analyse by user maintained groups in column three). Select a ‘From’ and ‘To’ date, optional filters include ‘Minimum Values’, ‘Number of’ (quantity) & ‘Total Value’ in currency, e.g. 1 = £1 or €1. The report can be produced in ‘Detail’ or ‘Summary’ & the results can be filtered by ‘Mail Shot flags’ removing anyone from the list with a flag not to be mailed & ‘Hide Closed Accounts’ ensuring customer accounts that are closed are not included in the report.

Select the office(s) to analyse, to select all of them click on the ‘Select Offices to Analyse or Click for All’ button. Next – select the section type to analyse by, to select all of them click on the ‘Select Sections to Analyse or Click for All’ button. The user can drill down within a section by ticking the ‘Section Breakdown’ text box and then selecting .

Select the required ‘Types’ within a section by clicking the box next to the description. Further filtering can be doen by tciking the ‘Weight Range’ box and entering a ‘From’ and ‘To’ range. Click F5 or the ‘Continue’ button to proceed.

In addition, further filtering can be done by entering a range of postcodes, tick the ‘Select By Postcode’ then enter the first characters of a postcode area and press the enter key, the postcode entered will drop into the bottom text box, the user can enter several postcodes, additonal options include ‘Exclude blank postcodes’.

Additional options include mailing types & sub groups/breeds.

Click on the ‘Run Analysis’ button

Double click any of the grid headings to sort by, e.g. to view the highest value double click on the ‘value’ column. The following options are available; Save Report, Export CSV, Print Address Lables, SMS Text, Print Report, Sage Export, Internet Fax, Email & ‘Multi-Quer y’.

Double clicking on grid will load the ‘View / Amend Customer’ routine. Right clicking on the grid will provide  the following options;

‘View All’ drills down to each transaction relating to the customer trading history.

Double click on a transaction in the grid to drill down to the complete transaction history for the lot.

‘View All Grouped By’ provides filtering options by; Buyer, Date, Item Type, Grade, Date + Buyer & Date + Seller. ‘Delete from List’ will remove the customer from the current report.

If the ‘View/Add Comments’ button is showing it means there is a comment on the transaction, if the button is labelled Add Comments the user can add comments by clicking on the button but currently there are no comments.

Multi-Queries

Select the required parameters for the trading analysis. With the results on screen click on the ‘Save Results’ button in the Multi-Query window.

The user will be prompted that the results have been saved. Press escape on the keyboard and enter the second date range to search through and click on ‘Run Analysis’. Click on Save Results again and then click on ‘View Merged’ and the data from both searches will merge into one result of data.

When finished with the merged data click on the ‘Clear’ button and then click on ‘Yes’ the results will be disguarded.

Address Labels

Click on the ‘Print Address Labels’ button. The user can allocate the routine to the contact management suite, allowing further analysis. Click no to bypass the contact management tie-up.

Text Messaging

The system will automatically use the mobile numbers stored against the customer account and will populate the list. Type in the required message in the ‘SMS Text Message’ textbox. Click on the ‘Add Mobile No’ button and add as many mobile numbers as required. to delete a mobile number from the list click on it and press the delete key on the keyboard. To send the texts click on ‘Send SMS Texts’ button. The user will need to have setup an account with Esendex at www.esendex.com

Click on ‘Export to CSV’ to export the mobile numbers to a csv file for third party use.

NAS also has links with www.textlocal.com

Email

The system will automatically use the email addresses stored against a customers account from the search results that have been created. Add a subject for the message and add the message itself. Click on the browse button if an attahcment is required. add emails by clicking on the ‘Add email’ button and delete emails from the list by clicking on them and pressing the delete key on the keyboard. Click on the ‘Send Email’ button when ready to proceed.

Seller / Buyer Non Trading Analysis

Description – allows the user to search through trends of customer that have ceased trading with the business over a user defined period.

It works in exactly the same way as the trading analysis except the user specifies the dates of trading and non trading

History Search

Description – Is designed to allow the user to search through all closed sales for any information.The search database can be accessed by the following three options, Lot Number Search, Lot Description Search & Ear Tag Search.

Click on the icon of the calendar to set a ‘start date’, do the same for an ‘end date’. Select the search criteria by clicking on the appropriate button, ‘Ear Tag Number’ (Cattle Ear Tags), ‘Lot Number’, ‘Remarks’ (will search the short description entered against a lot), ‘Extra Description’ (will search the catalogue description entered against a lot) & ‘Sheep Ear Tag No’. Filtering can be achieved by the following search options; ‘Contains’ (searching for a word or number containing the search entered), ‘Begins with’ (will search for only words or numbers beginning with the entered search) & ‘Equals’ (will only bring back exact matches of the entered search). In the example, lot number 100 has been entered.

All lot 100 will be returned for the date range entered.

Double click on a lot to drill down to the full transaction.
